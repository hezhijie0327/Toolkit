# Toolkit 工具集

一个用于自动化 Git 和 GPG 操作的 Shell 脚本工具集。

## 简介

Toolkit 是一个基于 Bash 的轻量级工具集，提供了自动化 Git 和 GPG 操作的脚本。无需外部依赖，仅需标准的 Unix 工具即可运行。

## 安装

```bash
git clone "https://github.com/hezhijie0327/Toolkit.git"
```

## 使用方法

### Git 操作脚本 (Git.sh)

#### 克隆仓库

```bash
bash ./Git.sh -u "用户名" -r "仓库名" -b "分支名" -d "深度" -m "clone"
```

**示例：**

```bash
bash ./Git.sh -u "hezhijie0327" -r "Toolkit" -b "main" -d "1" -m "clone"
```

#### 提交并推送更改

```bash
bash ./Git.sh -u "用户名" -r "仓库名" -e "邮箱地址" -f "文件路径" -i "提交信息" -m "push"
```

**示例：**

```bash
bash ./Git.sh -u "hezhijie0327" -r "Toolkit" -e "hezhijie0327@hotmail.com" -f "." -i "Generated by GitHub Actions" -m "push"
```

**参数说明：**

- `-u`: GitHub 用户名
- `-r`: 仓库名称
- `-b`: 分支名称（仅克隆模式）
- `-d`: 克隆深度（仅克隆模式，数字）
- `-e`: 用户邮箱（仅推送模式）
- `-f`: 要提交的文件路径（仅推送模式）
- `-i`: 提交信息（仅推送模式）
- `-m`: 操作模式（clone 或 push）

### GPG 操作脚本 (GPG.sh)

#### 解密文件

```bash
bash ./GPG.sh -m "decrypt" -p "密码" -f "文件路径"
```

#### 加密文件

```bash
bash ./GPG.sh -m "encrypt" -r "收件人" -f "文件路径"
```

#### 导出密钥

```bash
# 导出私钥
bash ./GPG.sh -m "export" -t "private" -p "密码" -k "密钥文件路径"

# 导出公钥
bash ./GPG.sh -m "export" -t "public" -r "用户ID" -k "密钥文件路径"
```

#### 导入密钥

```bash
# 导入私钥
bash ./GPG.sh -m "import" -t "private" -p "密码" -k "密钥文件路径"

# 导入公钥
bash ./GPG.sh -m "import" -t "public" -k "密钥文件路径"
```

#### 签名文件

```bash
bash ./GPG.sh -m "sign" -e "编码格式" -p "密码" -f "文件路径"
```

**编码格式选项：**

- `asc`: ASCII 装甲分离签名
- `gpg`: GPG 二进制签名
- `sig`: 二进制分离签名

#### 验证文件

```bash
bash ./GPG.sh -m "verify" -f "文件路径" -s "签名文件路径"
```

**参数说明：**

- `-m`: GPG 操作模式（decrypt, encrypt, export, import, sign, verify）
- `-p`: 密码（某些操作需要）
- `-f`: 文件路径
- `-r`: 收件人/用户ID
- `-k`: 密钥文件路径
- `-s`: 签名文件路径
- `-t`: 密钥类型（private 或 public）
- `-e`: 编码格式（asc, gpg, sig）

## 系统要求

- Bash shell
- Git（用于 Git 操作）
- GPG（用于 GPG 操作）

## 错误处理

所有脚本都包含完整的参数验证和错误处理：

- 缺少必需参数时会显示友好的错误信息
- 无效的参数值会被检测并报告
- 文件操作前会检查文件是否存在
- 操作成功或失败都会给出明确反馈
